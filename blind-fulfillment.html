Orders

Field Required? Description Options
shipment_id No This field is used to track the order through the whole process. It ties an order to items within the shipment. The import system will supply the shipment_id. Leave blank
order_id Yes This is usually an order number/ID that is used as a reference back to the originating shopping system (shopify, magento, woocommerce...etc). In the event the originating system needs an export we can tie our system to the original source system. If an order ID is not provided please see the notes in the Legend section. Legend options
merchant_id Yes This is an ID number to reference the merchant account associated with this fulfillment.

currency Yes The 3 letter currency code the order was sold through. This is necessary when it comes to CN22 forms where we have to declare a value to the shipment.
  • USD
  • EUR
  • GBP
total Yes The retail value of the whole shipment. Necessary when it comes to the CN22 forms where we have to declare a value to the shipment. This should be a decimal value with NO currency symbol in the front. e.g.: 5.00 or 100.35
order_email Yes This is the customer's email address. e.g.: tobiasFunke@gmail.com
shipping_name Yes The full name of the customer. e.g.: Tobias Funke
shipping_company No The company name or "c/o" line of the customer. e.g.: c/o Gobias Industries LLC
shipping_address1 Yes The address line 1 of the customer's shipping address. This should not include unit or apartment numbers. e.g.: 123 Sudden Valley Way
shipping_address2 Maybe The address line 2 of the customer's shipping address. This usually is for unit or apartment numbers. It is required if the address has an Address Line 2. e.g.: Unit 1337
shipping_city Yes The city of the customer's address. e.g.: Newport Beach
shipping_zip Yes The zip/postal code of the customer's address. e.g.: 92661 or 92661-1324 (zip + 4)
shipping_province Yes State or province of the customer's address. If within the US it needs to be the 2 letter abbreviated state code. e.g.: CA, FL, GA, RI, NY, NJ...etc
shipping_country Yes The ISO 3166 country code for the country of the customer's address. Get the ALPHA-2 ISO codes from here: ISO Codes. e.g.: US, CA, GB, DE...etc
shipping_phone Maybe The customer's phone number. This is used by the shipment service if they need to reach the customer. If no phone number is provided please use our District Lines support number. e.g.: 1-877-739-1173
shipping_method Yes The numerical value of the method this shipment should go out as. These are predefined shipment types. If the shipment method is not present, please contact the Development team to discuss.

Items

You will create at least one row per order. If the shipment includes multiple items for pickers, then create one row per item. The order_id is used to combine an order with all the shippable items.
Field Required? Description Options
id No This field should be left blank. The import system will generate this value. Leave blank
shipment_id No This will match the shipment_id from the order. Once the order is imported into the fulfillment system this value will be provided. Leave blank
order_id Yes This value should match the order_id field from the order sheet which ties together this item or items with the order information. -
qty Yes The amount to be packed for this item. Numerical whole number. e.g.: 1
name Yes The name of the item to be packed. This shows on the packing slip. e.g.: Magic Is For Real Crewneck
price Yes The retail value of the item. Necessary when it comes to the CN22 forms where we have to declare a value to the shipment. This should be a decimal value with NO currency symbol in the front.
sku No The unique SKU number for the item. e.g.: 19441-29-63
size Yes The size of the item. e.g.; Small, Hat, Book...etc
image No The file name of the item image. Useful if the item is very similar and a photo on the packing slip will help packers pick the correct item. Please supply the image file with the Development team. e.g.: gob-magic-is-real.jpg

Legend

This section is used to generate an order number for the shipments if the originating data source does not have order numbers. This helps us tie our fulfillment system to the originating source's system.
Only change the 2nd row of data on the Legend sheet. The rest of the rows are already setup with functions to generate the appropriate format.
When you generate the order_id you can use the Excel functions to copy them over to the appropriate Orders and Items sheet. The template has an example of this being done on the Order sheet.
Field Description
prefix This will be the prefix to the order number. This needs to be either an alpha numeric character.
It cannot start with a 0 or contain any special characters (i.e.: ~`-=[]\;',./!@#$%^&*()_+{}:"<>?|)
numbering Enter a numeric value that this number should start at. By default it will start with one. If you want it to start at 355 then type 355 in the 2nd row.
combined Do not touch - This is is an Excel formula that will generate the final order number in sequence.
